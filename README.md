## 快速开始

### 环境要求

- WSL (Ubuntu 22.04/24.04) 或 Linux
- ARM交叉编译器: `gcc-arm-linux-gnueabihf`
- QEMU用户模式模拟器: `qemu-user-static`

### 安装依赖

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install gcc-arm-linux-gnueabihf qemu-user-static
```

### 编译运行

```bash
cd exp_benchmark

# 编译
arm-linux-gnueabihf-gcc -static -o qemu_test qemu_test.c -lm

# 运行
qemu-arm-static ./qemu_test
```

## 算法介绍

### 1. CMSIS-NN (推荐)

**来源**：ARM官方CMSIS-NN库

**方法**：多项式近似 + 查找表

**特点**：
- 原始函数 `arm_nn_exp_on_negative_values()` 为softmax设计
- 输入格式：Q5.24，输出格式：Q16.16
- 精度最高，周期最少
- **纯定点运算，无浮点依赖**

```c
#include "cmsis_nn_exp.h"

int32_t result = cmsis_nn_exp_q524(x_q524);  // 输入Q5.24，输出Q16.16
```

### 2. CORDIC

**方法**：双曲坐标系迭代旋转

**特点**：
- 25次迭代（含重复迭代i=4,13）
- 定点格式：Q24输入，Q16.16输出
- 仅需移位和加法，无需乘法器
- 硬件友好，适合FPGA/ASIC实现
- **纯定点运算，预计算查找表**

```c
#include "cordic_exp.h"

int32_t result = cordic_exp_q24(x_q24);  // 输入Q24，输出Q16.16
```

### 3. Haetae

**来源**：参考pqm4后量子密码库haetae签名算法的原理

**方法**：Taylor展开（8项）+ 查找表

**注意**：本实现为纯C语言版本，非pqm4原始汇编代码

**特点**：
- 16位小数精度
- 最大误差最低（0.1221%）
- 适合密码学应用
- **纯定点运算，整数部分使用查找表**
- 支持正负值 [-10, 10]

```c
#include "haetae_exp.h"

int32_t result = haetae_exp_q16(x_q16);  // 输入Q16.16，输出Q16.16
```

## 适用场景

| 场景 | 推荐算法 | 理由 |
|------|----------|------|
| 神经网络推理 | **CMSIS-NN** | 最快，ARM官方支持，精度最高 |
| FPGA/ASIC实现 | **CORDIC** | 无需乘法器，硬件友好 |
| 密码学应用 | **Haetae** | 最大误差最低 |
| 通用嵌入式 | **CMSIS-NN** | 性能最优 |
